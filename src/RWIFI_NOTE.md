# ⚠️ Важное замечание о файле rwifi.h

## Почему файл назван rwifi.h?

Файл WiFi модуля намеренно назван **`rwifi.h`** (и `rwifi.cpp`), а не `wifi.h`, по следующей причине:

### Проблема

В системе Arduino ESP32 существует системный заголовочный файл **`WiFi.h`**, который подключается директивой:
```cpp
#include <WiFi.h>
```

Если ваш проект содержит файл с именем `wifi.h`, компилятор может подключить его вместо системного файла, что приводит к ошибкам:
- `'WiFi' was not declared in this scope`
- Отсутствуют определения классов и функций

### Решение

Файл назван **`rwifi.h`** (robot WiFi) для:
1. Избежания конфликта имён с системным `WiFi.h`
2. Чёткого разделения пользовательского кода и библиотек
3. Упрощения отладки и понимания структуры проекта

### Структура WiFi модуля

```
src/
├── rwifi.h     ← Заголовочный файл (ваш код)
├── rwifi.cpp   ← Реализация (ваш код)
└── ...
```

При подключении в других файлах используется:
```cpp
#include "rwifi.h"  // Ваш модуль
#include <WiFi.h>   // Системная библиотека ESP32
```

---

**Запомните:** При изменении настроек WiFi редактируйте файл `src/rwifi.cpp`!
