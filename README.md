# Документация проекта: ESP32-S3 Robot Controller

## 📋 Обзор проекта

Проект представляет собой систему управления роботом на базе микроконтроллера **ESP32-S3**, обеспечивающую управление сервоприводами через драйвер PCA9685 и DC-моторами с танковым управлением.

### Основные возможности

- 🎯 Управление до 16 сервоприводами через I2C-драйвер PCA9685
- 🚜 **Танковое управление** 4 DC-моторами (A, B, C, D)
- 📡 **WiFi управление** через веб-интерфейс и REST API
- 🕹️ **Виртуальный джойстик** с 3 режимами (Езда, Серво, Микс)
- 🔄 **OTA обновления** прошивки через WiFi
- 💬 Интерфейс командной строки через Serial
- ⚙️ Гибкая настройка параметров сервоприводов

## 🔌 Аппаратная часть

### Микроконтроллер
- **Плата:** ESP32-S3-DevKitC1-N16R8
- **Фреймворк:** Arduino
- **Тактовая частота мониторинга:** 115200 бод

### Сервоприводы (PCA9685)
| Параметр | Значение |
|----------|----------|
| I2C SDA | GPIO 8 |
| I2C SCL | GPIO 9 |
| Адрес драйвера | 0x40 |
| Частота PWM | 50 Гц |
| Разрешение | 12 бит (0-4095) |
| Каналы | 16 (используются 0-3) |

### Конфигурация робота

**Сервоприводы и моторы:**
| Серво | Мотор | Сторона | Назначение |
|-------|-------|---------|------------|
| 0 | B | Левая | Поворот влево/вправо |
| 1 | A | Правая | Поворот влево/вправо |
| 2 | D | Правая | Наклон вперёд/назад |
| 3 | C | Левая | Наклон вперёд/назад |

**DC-моторы (GPIO):**
| Мотор | Вперёд | Назад | Сторона |
|-------|--------|-------|---------|
| A | GPIO 14 | GPIO 13 | Правая |
| B | GPIO 12 | GPIO 11 | Левая |
| C | GPIO 18 | GPIO 17 | Левая |
| D | GPIO 16 | GPIO 15 | Правая |

### Используемые библиотеки
- **Adafruit PWM Servo Driver Library** v3.0.2+ — управление PCA9685
- **ArduinoJson** v7.0.4+ — парсинг JSON

## 📖 Описание модулей

### 1. `main.cpp` — Главный модуль

Точка входа в программу. Выполняет:

- Инициализацию сервоприводов и DC-моторов
- Основной цикл обработки HTTP запросов

### 2. `pins.h` — Конфигурация пинов

Определяет назначения GPIO пинов:

```cpp
// I2C для PCA9685
#define I2C_SDA 8
#define I2C_SCL 9

// Мотор A (правый)
#define A_IA 14  // Вперёд
#define A_IB 13  // Назад

// Мотор B (левый)
#define B_IA 12  // Вперёд
#define B_IB 11  // Назад

// Мотор C (левый)
#define C_IA 18  // Вперёд
#define C_IB 17  // Назад

// Мотор D (правый)
#define D_IA 16  // Вперёд
#define D_IB 15  // Назад
```

### 3. `servo.h/cpp` — Управление сервоприводами

#### Структуры и константы

**ServoConfig** — структура настроек сервопривода:
```cpp
struct ServoConfig {
  uint16_t currentAngle;   // Текущий угол (0-180°)
};
```

**Константы:**
- `MAX_SERVOS` = 16 — максимальное количество сервоприводов
- `SG92R_PWM_MIN` = 140 — минимальное значение PWM для SG92R
- `SG92R_PWM_MAX` = 480 — максимальное значение PWM для SG92R
- `correction[]` — массив коррекции углов для каждого сервопривода

#### Функции

##### `void setup_serv()`
Инициализирует систему управления сервоприводами:
1. Инициализация I2C (GPIO 8, 9)
2. Инициализация PCA9685
3. Установка частоты PWM 50 Гц
4. Инициализация массива конфигураций
5. Установка всех сервоприводов в положение 90°

##### `void servo_setAngle(uint8_t servoNum, uint16_t angle)`
Устанавливает угол поворота сервопривода (для API):
- **servoNum** — номер канала (0-15)
- **angle** — угол в градусах (0-180)
- Применяет коррекцию из массива `correction[]`

##### `void servo_setLimits(uint8_t servoNum, uint16_t newMin, uint16_t newMax)`
Настраивает границы импульсов для сервопривода (для API).

##### `uint16_t servo_getAngle(uint8_t servoNum)`
Возвращает текущий угол сервопривода.

##### `uint16_t servo_getMin(uint8_t servoNum)` / `servo_getMax(uint8_t servoNum)`
Возвращают минимальное/максимальное значение импульса.

---

### 4. `dcmotor.h/cpp` — Управление DC-моторами

#### Функции

##### `void setup_dc()`
Инициализирует управление DC-моторами:
- Настраивает GPIO пины как выходы
- Останавливает все моторы

##### `void motor_setSpeedA/B/C/D(int speed)`
Устанавливает скорость соответствующего мотора (-255...255).

##### `void motor_getSpeedA/B/C/D()`
Возвращают текущую скорость моторов.

##### `void motor_stopAll()`
Останавливает все моторы.

---

### 5. `rwifi.h/cpp` — WiFi подключение

Подключение к существующей WiFi сети:
- SSID и пароль задаются в `rwifi.cpp`

## 📡 WiFi Управление

Проект включает полнофункциональную систему WiFi управления с веб-интерфейсом и REST API.

### Быстрый старт WiFi

1. **Подключитесь к WiFi сети робота:**
   - SSID: ``
   - Пароль: ``

2. **Откройте веб-интерфейс:**
   - URL: `http://<IP-адрес>:8080`
   - Управление сервоприводами и моторами через браузер

3. **Используйте REST API:**
   - Базовый URL: `http://<IP-адрес>:8080/api`
   - Интеграция с внешними приложениями

### Веб-интерфейс

**Две вкладки:**

#### 🎛️ Параметры
- Ручное управление 4 сервоприводами (0-180°)
- Калибровка мин/макс границ для каждого серво
- Управление 4 DC-моторами:
  - Кнопки: ◀ (назад), ■ (стоп), ▶ (вперёд)
  - Ползунки точной настройки (-255...255)
  - Панель управления всеми моторами сразу

#### 🕹️ Джойстик
**3 режима работы:**

1. **🚗 Езда** — танковое управление моторами:
   - X ось — поворот (левые/правые моторы в противоположных направлениях)
   - Y ось — движение вперёд/назад
   - Левые моторы: B, C
   - Правые моторы: A, D

2. **🔧 Серво** — управление сервоприводами:
   - X ось — поворот корпуса (левые и правые серво в противоположных направлениях)
   - Y ось — наклон корпуса (все серво в одном направлении)
   - Левые серво: 0 (B), 3 (C)
   - Правые серво: 1 (A), 2 (D)

3. **🎯 Микс** — одновременное управление моторами и серво

### REST API Эндпоинты

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/api/status` | Статус системы |
| GET | `/api/servo` | Получить сервоприводы |
| POST | `/api/servo` | Установить угол |
| GET | `/api/motor` | Получить моторы |
| POST | `/api/motor` | Установить скорость |
| POST | `/api/motor/stop` | Остановить все |

### Примеры API запросов

**Установка угла сервопривода:**
```bash
curl -X POST http://192.168.4.1:8080/api/servo \
  -H "Content-Type: application/json" \
  -d '{"id":0,"angle":90}'
```

**Управление моторами:**
```bash
curl -X POST http://192.168.4.1:8080/api/motor \
  -H "Content-Type: application/json" \
  -d '{"motorA":100,"motorB":100}'
```

## 🛠️ Расширение функциональности

### Настройка коррекции сервоприводов

В файле `servo.cpp` массив `correction[]` позволяет подстроить каждый сервопривод:

```cpp
int correction[MAX_SERVOS] = {-5, -13, -8, -20};
```

Значения добавляются к углу перед отправкой на сервопривод.

### Добавление новых сервоприводов

Проект поддерживает до 16 сервоприводов. Для использования дополнительных каналов:
1. Используйте номера каналов 4-15 в командах
2. При необходимости измените значения `SG92R_PWM_MIN` и `SG92R_PWM_MAX` в `servo.cpp` для конкретных моделей

Настройте через команду `L<servo>,<min>,<max>` или измените константы в коде.

## ⚠️ Важные замечания

1. **Питание сервоприводов:** PCA9685 должен получать отдельное питание 5V, не полагайтесь только на питание от ESP32.

2. **Частота PWM:** Установлена на 50 Гц — стандарт для большинства сервоприводов.

3. **Диапазон углов:** Ограничен 0-180° программно для защиты механизмов.

4. **I2C адреса:** По умолчанию используется адрес 0x40. При изменении адреса обновите в `servo.cpp`.

5. **Скорость моторов:** Диапазон -255...255, где:
   - Положительное значение — вперёд
   - Отрицательное — назад
   - 0 — остановка

6. **Python версия:** PlatformIO требует Python 3.10–3.13 (Python 3.14 не поддерживается!)

## 🐛 Отладка

### Включение отладочного режима

Раскомментируйте в `platformio.ini`:
```ini
build_type = debug
debug_speed = 40000
debug_tool = esp-builtin
debug_build_flags = -O0 -g -ggdb
```

### Логирование

**Serial (115200 бод):**
- `[API]` — запросы к REST API
- `[OTA]` — процесс OTA обновления
- `[UI]` — раздача статических файлов

**Консоль браузера (F12):**
- `[Joystick]` — команды джойстика
- `[Joystick Servo]` — управление сервоприводами

### Типичные проблемы

| Проблема | Решение |
|----------|---------|
| PCA9685 не найден | Проверьте подключение I2C и адрес 0x40 |
| Сервопривод дрожит | Проверьте питание, возможно требуется внешний источник |
| Неправильные углы | Настройте MIN/MAX через команду L или массив correction[] |
| Нет ответа в Serial | Проверьте скорость 115200 и подключение USB |
| Джойстик не работает | Проверьте загрузку joystick.js в консоли браузера |
| Python 3.14 ошибка | Установите Python 3.10–3.13 |
